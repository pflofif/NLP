## 6. Error analysis

Всього помилок на test: **0** (FP=0, FN=0)

B2 класифікував усі 83 пари правильно на тестовій вибірці. FP = 0 пояснюється малим позитивним класом (3 пари) і сильним сигналом від noun\_adj-токенів. Реальні помилкові патерни очікуються при ширшому покритті запитів:

- **FP:** overlap тематик — схожа лексика між різними закладами
- **FN:** рідкісна лексика / сленг — запит містить терміни, відсутні в train
- **FN:** короткий відгук — слабкий сигнал для класифікатора

## 7. Що фіксити далі

- Hard negative sampling (тематично близькі, але нерелевантні документи)
- Threshold tuning на val set для зменшення FP при розширенні датасету
- BM25 як альтернативний baseline для порівняння з TF-IDF
- Збільшити кількість анотованих запитів (зараз 40 — замало для надійного тесту)
